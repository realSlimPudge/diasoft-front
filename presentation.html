Наше решение

Единый путь для трёх участников

01
🏛️ ВУЗ загружает реестр
CSV или XLSX файл с дипломами. Система принимает, проверяет, обрабатывает асинхронно. Видны ошибки по строкам.
02
🎓 Студент получает доступ
QR-код для своего диплома. Временные share-ссылки для работодателей — с TTL от 1 часа до 30 дней.
03
💼 Работодатель проверяет
Форма, QR-сканер или прямая ссылка. Мгновенный ответ: valid / revoked / expired. Без авторизации.
04
⚡ Публично и быстро
Проверка через Redis-кэш. Публичный контур не нагружает внутренний реестр. Работает на миллионах дипломов.

Как достигаем масштаба
  • Публичная проверка отделена от реестра — нагрузки не пересекаются
  • Redis кэш на горячих верификациях — без поиска по всей базе
  • Kafka outbox — registry и gateway общаются асинхронно
  • Async import worker — большие файлы обрабатываются по частям
Production roadmap (следующий этап)
  • Session-based upload: браузер → S3 напрямую (presigned URL)
  • Normalizer + parallel chunk-workers вместо одного воркера
  • PostgreSQL partitioning для таблиц дипломов и импортов
  • Signed QR — цифровая подпись ВУЗа вместо read-model lookup
Модель доверия

От lookup к цифровой подписи

Сейчас · Working
📋 Проверка по read-model
Gateway нормализует запрос → проверяет Redis кэш → смотрит в verification_records (PostgreSQL projection из Kafka). Быстро, работает прямо сейчас.
✅ SHA-256 record_hash
Реестр вычисляет и хранит детерминированный хэш полей диплома. Используется для аудита и integrity-проверок. Хранится в source-of-truth и в Kafka событиях.
🚫 Отзыв работает сейчас
После revoke через кабинет ВУЗа — статус обновляется в реестре, событие уходит в Kafka, gateway projection обновляется, публичная проверка возвращает verdict=revoked.

Roadmap
Цель · Signed QR
🔐 Цифровая подпись ВУЗа
QR содержит подписанный payload диплома. Подпись верифицируется публичным ключом ВУЗа — аутентичность не зависит от read-model.
⚡ Один indexed lookup
После проверки подписи — один быстрый запрос по ключу для актуального статуса (отозван / не отозван). Работает на 3M+ дипломах без деградации.
🛡️ Защита от подделки
Данные диплома нельзя изменить без private key ВУЗа. Скомпрометированный read-model не может создать "валидный" диплом.
